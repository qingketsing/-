## Redis基本定位

1. Redis是什么？
	Redis 是一个基于内存的高性能 key-value 数据库，支持多种数据结构，通常用在分布式锁，缓存，计数器等高并发场景。
2. Redis和MYSQL的区别
	MYSQL是关系型数据库，主要负责长期可靠而且结构化的数据存储。Redis是内存型的kv存储，主要负责高频访问，低延迟读写。Redis的访问速度远快于MYSQL，但不适合复杂关系查询。
3. Redis为什么快？
	基于内存，避免磁盘IO；单线程执行，没有过多的互斥锁操作和上下文开销；IO多路复用，不需要一个连接一个进程；数据结构优化，跳表，哈希等。
4. Redis的单线程模型
	第一是缓存，比如缓存用户信息、商品详情、配置数据，减少数据库压力。
	第二是分布式锁，比如用 `SET key value NX EX` 实现互斥控制，防止多个实例同时操作同一资源。	
	第三是计数器，比如点赞数、阅读量、接口访问次数。
	第四是排行榜，使用 ZSet 可以根据分数排序，比如积分榜、热度榜。
	第五是限流，比如用 String 或 Lua 脚本实现固定窗口、滑动窗口限流。
	第六是 Session 或登录态存储，在多台服务器之间共享用户状态。
5. IO多路复用
	Redis 在处理客户端命令时，核心命令执行逻辑是单线程的，同一时刻只由一个线程顺序执行客户端命令。好处是实现简单，避免了多线程加锁和上下文切换，竞态问题。Rediszhu'ya
