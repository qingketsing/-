## Redis基本定位

1. Redis是什么？
	Redis 是一个基于内存的高性能 key-value 数据库，支持多种数据结构，通常用在分布式锁，缓存，计数器等高并发场景。
2. Redis和MYSQL的区别
	MYSQL是关系型数据库，主要负责长期可靠而且结构化的数据存储。Redis是内存型的kv存储，主要负责高频访问，低延迟读写。Redis的访问速度远快于MYSQL，但不适合复杂关系查询。
3. Redis为什么快？
	基于内存，避免磁盘IO；单线程执行，没有过多的互斥锁操作和上下文开销；IO多路复用，不需要一个连接一个进程；数据结构优化，跳表，哈希等。
4. Redis的单线程模型
	Redis 在处理客户端命令时，核心命令执行逻辑是单线程的，同一时刻只由一个线程顺序执行客户端命令。好处是实现简单，避免了多线程加锁和上下文切换，竞态问题。Redis主要操作都是在内存中进行的，CPU通常不是瓶颈，瓶颈主要是内存和网络IO，单线程反而可以保证较高的性能。
5. IO多路复用
	一个线程可以同时监听多个网络连接上的IO事件。这样 Redis 不需要为每个客户端创建线程，也能处理高并发连接。
6. kv存储模型
	通过一个唯一的 key 去访问对应的 value。Redis 中所有数据都是围绕 key 组织的，value 可以是 String、Hash、List、Set、ZSet 等不同数据结构。Redis 的 key 设计很重要，一般会用业务前缀加 ID 的方式，比如 `user:1001`、`order:20260603:888`。这样可读性好，也方便管理和排查问题。
