# Golang 内存逃逸
#Golang
1. 什么是内存逃逸？
	内存逃逸是指函数内部创建的变量或者对象，在函数结束后仍然被其他部分引用或者持有
2. 堆栈的区别：
	栈上的内存分配速度快但是空间有限，由编译器自动管理分配和释放。
	堆上内存分配需要runtime的参与，空间大但是运行速度较慢，由GC机制回收。
3. 逃逸的影响有什么？
	分配和回收导致额外的开销，指针引用和内存安全，涉及内存泄漏的风险
4. 导致内存逃逸的原因是什么？
	栈空间大小是否足够，作用域是否变化，编译时无法确定类型或大小，指向栈上对象的指针不能存储到堆，其生命周期也不能超过所指对象的生命周期
5. 常见的内存逃逸的情况:

```Golang
// 第一种情况
// 指针，切片和map作为返回值
func f1()(*int, []int, map[int]int){
	i := 0
	list := []int{1,2,3,4}
	mp := map[int]int{1: 1, 2: 2}
	return &i, list, mp
}

// 第二种情况
// 向chan中发送数据或者包含指针的值
func f2(){
	ch := make(chan *int, 2)
	i := 2
	ch <- &i
	<- ch
}

// 第三种情况
// 非直接的函数调用，比如在闭包中使用闭包外的值
func f3() func(){ // 这里是指返回一个函数
	i := 0
	return func(){
		log.Println(i)
	}
}

// 第四种情况
// 切片或者map中存了包含指针的值
func f4(){
	i := 1
	list := make([]*int,5)
	list[0] = &i
}

// 第五种情况
// interface为多态应用的时候可能会直接在堆上分配内存
type animal interface{
	run()
}

type dog struct{}

func (d dog)run(){}

func f5(){
	// 以下这种不会逃逸，因为它是dog的明确类型
	var a animal = dog{}
	a.run()
	// 以下这种会逃逸，因为在声明变量的时候不知道是什么类型
	var b animal
	b = dog{}
	b.run()	
}
```
